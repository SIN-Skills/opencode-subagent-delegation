# Proactive Message Templates

## Periodic heartbeat
"Codex proactive checkpoint: report progress, then challenge your current approach for logic flaws, objective mismatch, and overengineering. Use owner labels Codex|Opencode|Joint."

## Critical pair checkpoint
"Codex critical pair check: before further execution, run counter-critique on current plan and return decision=accept|rework|escalate with rationale."

## Canonical drift intervention
"Codex intervention: canonical docs/contracts changed. Reconcile against canonical priority, challenge prior assumptions, and re-approve plan before execution."

## Contract/OpenAPI intervention
"Codex intervention: contract/openapi drift detected. Stress-test current implementation logic against latest contract, list impact, and re-order tasks."

## Rule conflict intervention
"Codex intervention: rule conflict signal detected. Pause implementation, resolve by priority, document critique + decision (accept|rework|escalate), then continue."

## Logic gap intervention
"Codex intervention: logic_gap suspected. Identify contradiction, provide concrete correction options, and update plan only after mutual critique closure."

## Scope drift intervention
"Codex intervention: scope drift detected. Compare against target outcome, remove non-essential work, and propose minimal high-impact path."

## Overengineering intervention
"Codex intervention: overengineering signal. Propose simpler architecture delivering same goal, include tradeoffs and decision."

## Risk escalation intervention
"Codex intervention: risk escalation triggered. Stop unsafe path, produce mitigation options, assign owners, and restart from approved replan."

## Parallelization boost
"Codex optimization: split independent workstreams, assign owners, and require cross-review between streams before merge."
